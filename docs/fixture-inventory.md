Current analysis observations:

- OAAdapter primary service:
  `oa.adapter.geographicAddressManagement:createGeographicAddressValidation`
- OAAdapter remains a normal FLOW service fixture: its `node.ndf` declares `svc_type=flow`.
- OAAdapter signature contains one `input` document reference and one `output` document reference.
- OAAdapter observed FLOW tags include `FLOW`, `SEQUENCE`, `BRANCH`, `LOOP`, `INVOKE`, `MAPINVOKE`,
  `MAP`, `MAPCOPY`, `MAPSET`, `MAPDELETE`, `MAPSOURCE`, `MAPTARGET`, `DATA`, and `EXIT`.
- OAAdapter static invoke-like steps extracted: 43.
- OAAdapter static invoke targets are unresolved in the local sample snapshot because corresponding
  package metadata for adapter, `pub.*`, and related targets is not present.
- PGP active FLOW Services parsed by the same parser: 23.
- PGP active FLOW Services include static `INVOKE` and `MAPINVOKE` calls, but the observed sample set
  does not include `LOOP` or `EXIT`.
- PGP includes 11 Java Services. M4a analyzes them from generated source and `java.frag` without
  executing, compiling, or loading Java code.
- PGP includes 7 active Document Type artifacts. OAAdapter includes service signature references to
  two OA document types, but those document artifacts are not present in the local snapshot.
- Current samples contain no real opaque/non-FLOW/non-Java service fixture. M5-lite production
  behavior for opaque services is therefore validated with synthetic common-`node.ndf` tests only.
  Detailed JDBC Adapter Service, SQL, connection-alias, UM/JMS, trigger, scheduler, and native
  process-model formats remain unproven by fixtures.
- Current samples contain no default `processes.yml`. M6 process catalog behavior is validated with
  synthetic and external test catalogs only; the default fixture analysis has zero declared process
  definitions.
